NAME

r.population.density.py - Creates a weighting layer with Random Forest for population redistribution

KEYWORDS

Land Cover, Density, Population, Dasymetry, Random Forest

SYNOPSIS

r.population.density.py
r.population.density.py --help

DESCRIPTION

Densite_population Creates a Random Forest model trained at the level of the administrative units and predict a weight for each grid of the output layer.

Random Forest's covariates are essentially the proportion of each Land cover's class.The response variable of the model is the log population density which represents the weight of each grid. A weight of 0 is set for each grid whose built-up proportion is null.

Other covariates can optionally be added to the model. It includes proportion of any other raster's class (input file: Land Use, Urban morphological areas... ) and distance to zones of interest (input file: raster whose grids represent distance to the nearst point of interest: school, hospital, university...).

The oob_score of the model will be printed in the console and the output files are a weighting layer and a plot representing the covariates importances.

NOTES

It is possible to not use all the classes whether for the Land Cover or for the optional raster in the Random Forest model. A list of classes to be used is then required.

Tasks are executed in parallel. That's why a size of the thread-pool is required. more information are about this parameter are here

Examples

Validation of population estimation

# set region and resolution
g.region raster=Land_cover res=gscript.raster_info(Land_cover)
# generate a weighting layer
Densite_population vector=admon_units Land_cover=Land_cover
					raster = morpho_refined tile_size=100
					id=Sect_ID population=Total area=Area
					built_up_category=11 output=weight
					png=F://importances_plot
